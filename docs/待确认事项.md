## 必须确认清单

1. 正式线索接入链路
   -  第一版正式链路是否确定为：`巨量 / GMP Webhook → auto_wechat` 
   -  同步拉取 douyinAPI 是否只保留为过渡 / 兜底 / 测试链路 
   -  如果 webhook 和同步拉取都保留，同一线索从两条链路进入时如何去重 
2. 正式 callback_url / webhook 地址
   -  正式回调地址最终使用哪个 
   -  是否继续使用 `callback.misanduo.com/webhook/douyin` 
   -  还是切换为 auto_wechat 自己的公网地址 
   -  callback_url 切换由谁操作 
   -  是否需要灰度切换和回滚方案 
3. 正式 webhook 接口契约
   -  正式路径 
   -  签名字段 
   -  签名算法 
   -  secret 配置方式 
   -  timestamp 过期窗口 
   -  成功返回格式 
   -  签名失败返回格式 
   -  重复事件返回格式 
   -  非线索事件返回格式 
4. 真实线索事件类型规则
   -  哪些事件一定创建 `DouyinLead` 
   -  哪些事件只记录事件、不创建线索 
   -  `im_enter_direct_msg` 是否默认视为有效线索 
   -  `latest_event`、`latest_scene`、`event_type` 以哪个字段为准 
5. DouyinLead 唯一键和去重规则
   -  是否使用数据源 `id` 作为 `external_lead_id` 
   -  如果 `id` 缺失，是否用 `open_id + account_open_id` 
   -  `conversation_short_id` 是否作为会话级辅助去重 
   -  `server_message_id` 是否只作为事件幂等字段 
   -  同一用户多次触发时，是更新原线索还是创建新线索 
6. 线索有效性规则
   -  没有手机号是否仍然有效 
   -  没有微信号是否仍然有效 
   -  只有 `open_id` 是否可以分配给销售 
   -  哪些字段缺失时标记 `invalid` 
   -  `invalid` 是否需要回调 
   -  `invalid` 是否展示在前端和导出数据中 
7. 真实线索字段映射
   -  `display_name` 映射到哪个业务字段 
   -  `lead_channel`、`lead_type`、`latest_event`、`latest_scene` 如何保存 
   -  `tags` 是否保存为 JSON 
   -  `raw_payload` 是否完整保存 
   -  `cleaned_payload` 是否保存 
   -  `avatar_url`、`conversation_short_id`、`server_message_id` 是否入库 
8. 对外回调接口文档
   -  回调地址 
   -  鉴权方式 
   -  签名方式 
   -  payload 格式 
   -  成功响应格式 
   -  失败响应格式 
   -  哪些错误码需要重试 
   -  最大重试次数是否固定 5 次 
9. 对外回调状态最终映射
   -  是否最终使用： 
     -  `received` 
     -  `invalid` 
     -  `assigned` 
     -  `agent_pulled` 
     -  `notified` 
     -  `send_failed` 
     -  `replied` 
     -  `timeout` 
     -  `reassigned` 
     -  `manual_required` 
     -  `failed` 
   -  是否确认 `callback_success` 只作为内部 `callback_logs.status = success`，不对外回调 
10.  **第一版账号初始化** 

-  第一位 `system_admin` 如何创建 
-  客户账号由谁创建 
-  客户初始密码如何交付 
-  是否第一版必须支持修改密码 
-  是否第一版需要重置密码 

1.  **LLM 失败策略** 

-  LLM 调用失败时进入 `check_failed` 还是 `detected_invalid` 
-  LLM 超时时间是多少 
-  连续失败几次后熔断 
-  熔断后是否只使用关键词判断 
-  LLM trace 脱敏字段范围 

1.  **前端第一版页面范围** 

-  登录页 
-  线索列表 / 详情 
-  销售管理 / 导入 
-  关键词配置 
-  工作时间配置 
-  超时 / 重分配配置 
-  Local Agent 状态 
-  微信任务列表 
-  回复检测列表 
-  回调失败列表 
-  人工处理页 
-  数据导出页 

1.  **人工处理规则** 

-  哪些状态允许人工处理 
-  人工处理后线索进入什么状态 
-  是否允许人工重新分配 
-  是否允许人工关闭线索 
-  是否允许人工补录销售回复 
-  是否允许人工标记回调已处理 

1.  **Excel 销售导入规则** 

-  销售姓名是否必填 
-  微信昵称是否必填 
-  手机号是否必填 
-  重复微信昵称是覆盖、跳过，还是报错 
-  是否支持部分成功 
-  是否返回错误行号和原因 

1.  **数据导出规则** 

-  第一版导出 Excel 还是 CSV 
-  导出是否脱敏 
-  是否支持按时间范围导出 
-  是否导出线索、分配记录、通知任务、回复检测、超时记录、回调失败记录、人工处理记录
