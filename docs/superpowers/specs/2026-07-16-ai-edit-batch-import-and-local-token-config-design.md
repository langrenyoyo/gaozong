# AI 剪辑批量导入与本机令牌配置修复设计

## 1. 目标

闭合本地开发环境中 AI 剪辑素材导入的两个问题：

1. 当前商户请求 `/api/ai-edit/agent-token` 返回 `LOCAL_AGENT_TOKEN_NOT_CONFIGURED`。
2. 素材库只能单选并导入一个视频，不能一次选择多个文件。

本轮不新增 19000 批量接口，不修改鉴权协议，不关闭 Local Agent 鉴权，不执行生产配置或宝塔验证。

## 2. 已确认根因

`.env.lan.local` 已配置 `LOCAL_AGENT_TOKENS`，但值只有 token，没有使用 9000 解析器要求的 `merchant_id:token` 格式。当前登录商户为 `m_nc_2bba00063cc13016`，因此 9000 无法找到商户对应 token，按合同返回 404。

Task 11 启动器输入的 token 只通过 `LOCAL_AGENT_TOKEN` 和 `LOCAL_AGENT_MERCHANT_ID` 注入 19000，不会回传给 9000。浏览器现有链路仍要求 9000 与 19000 使用同一 token。

## 3. 配置修复

保留 `.env.lan.local` 中现有 token，只将值原位改成：

```text
LOCAL_AGENT_TOKENS=m_nc_2bba00063cc13016:<现有token>
```

该文件不得提交。修改后重启本地 9000；启动测试 EXE 时输入相同 token。不得在日志、提交、文档或前端源码中写入 token 明文。

生产环境配置不在本轮执行，由后续生产验证窗口单独处理。

## 4. 批量导入交互

素材库继续使用现有隐藏文件选择框，增加多选能力。用户一次选择多个视频后：

1. 前端按选择顺序逐个调用现有 `importLocalMaterial` 流式接口。
2. 同一批次为每个文件生成不同的 `material_id`。
3. 单个文件失败时记录文件名和原因，继续导入后续文件。
4. 导入按钮显示 `导入中 当前数/总数`，执行期间禁止再次选择文件。
5. 批次结束后只刷新一次 9000 素材列表。
6. 全部成功显示成功数量；部分失败显示成功/失败数量和失败文件名；全部失败显示失败汇总。

顺序上传用于控制大视频对浏览器内存、本机磁盘和 19000 的瞬时压力。本轮不增加并发调度、暂停、续传或文件夹选择。

## 5. 错误处理与安全

- token 获取失败沿用现有错误处理，不回退到关闭鉴权。
- 401 仍清理当前商户会话缓存并重取一次 token。
- 每个文件继续使用 `application/octet-stream` 原始字节流，不转 Base64。
- 前端不自报可信 `merchant_id`，19000 仍通过 token 映射商户。
- 不在提示中展示 token、绝对路径或原始响应体。

## 6. 验收

1. 配置检查能确认 `LOCAL_AGENT_TOKENS` 含目标商户前缀，但不输出 token。
2. 前端合同门禁要求文件输入框具有多选能力，并禁止回退为仅处理 `files[0]`。
3. Playwright 使用两个临时视频文件，验证 19000 收到两次流式导入请求，页面不出现导入失败，且最终只刷新一次素材列表。
4. TypeScript 编译通过。
5. 前端生产构建通过。
6. 用户既有脏文件不纳入提交；`.env.lan.local` 不纳入提交。

## 7. 文档影响

实现完成后原位更新 Phase 12 AI 剪辑设计文档的素材导入验收事实；阶段状态、唯一 concern 和宝塔硬暂停口径不变。
