本设计扩展已落地的 Phase 12 本地 MVP，不回退 Task 0-11 的安全边界。目标是闭合素材回收站、自动分析、云端上传和素材库前端四个真实缺口。Task 12 不属于 Phase 13，不启动宝塔生产验证。

## 2. 当前事实与根因

### 2.1 回收站不可见

9000 `list_materials` 固定过滤 `deleted_at IS NULL`，但前端回收站依赖 `deleted_at` 非空记录。软删除虽已写入 `deleted_at` 与 `purge_after`，列表合同却永远不返回这些记录，因此回收站必然为空。

### 2.2 素材永久显示“处理中”

素材注册时默认写入 `analysis_status=pending`、`stabilization_status=pending`。当前导入链路没有自动分析队列，前端又把 `pending` 翻译为“处理中”，导致状态既不准确也不会收敛。

### 2.3 云端上传只有预留字段

模型已有 `storage_mode` 与 `cloud_storage_key`，但不存在云端上传接口、宝塔受控存储实现或前端入口。禁止只修改状态伪造上传成功。

### 2.4 前端信息结构不足

当前素材库只提供简单表格，缺少真实预览、分阶段状态、批量管理、分析结果、时间轴和云端操作。页面同时存在一个未被路由使用的旧同名占位组件，实施时不得误改该废弃入口。

## 3. 已批准需求

### 3.1 素材生命周期

- 本地导入为主，支持拖拽、多选文件和文件夹。
- 同商户按文件内容 SHA-256 去重；重复导入定位到既有素材。
- 删除后进入 7 天回收站，支持恢复和永久删除。
- 保留期内本地与云端副本都保留；永久删除或到期后同步清理。
- 活动剪辑任务引用存在时禁止破坏性清理。
- 视频可进入当前一键剪辑工作台；图片和音频在 Task 12 先完成导入、分析、预览、上传与生命周期管理，不扩展当前视频渲染或配乐协议。

### 3.2 自动分析

- 导入成功后自动开始分析。
- 输出媒体参数、语音转写、场景分段、口播/空镜分类、标签、高光、可用区间和稳定性评分。
- 媒体解析、语音转写、内容分析、稳定性检测、云端上传独立展示状态与进度。
- AI 结果允许人工修订；人工确认内容不被重新分析覆盖。
- 多模态服务不可用时保留本地结果，内容分析单独失败并可重试，禁止伪造结果。

### 3.3 多模态数据边界

- 原始视频只保存在 19000 本机受管目录，除用户主动上传云端外不得离开本机。
- 19000 只提取必要的低分辨率关键帧、转写和媒体参数。
- 9000 负责可信商户上下文并调用 9100；浏览器和 19000 不直连 9100。
- 9100 不接收本地路径、原始视频、密钥或不必要的媒体内容。
- `auto_edit` 的场景切分、关键帧、画质分析和适配器思路可迁入；其完整视频上传模型平台方案不采用。

### 3.4 增稳

- 自动检测稳定性，但不自动修改视频。
- 用户执行“一键增稳”后生成衍生素材，原素材不变。
- 增稳失败不得覆盖原素材或创建伪成功记录。

### 3.5 云端上传

- 默认仅保存在本机。
- 支持单个或批量手动上传到宝塔服务器的 9000 受控目录。
- 上传成功后继续保留本机副本。
- 必须真实流式写文件、校验大小与 SHA-256、原子提交并支持失败重试。

### 3.6 平台公共素材

- 普通商户只可查看、预览和用于剪辑。
- 上传、分析、修改和删除仅由平台管理员执行。

## 4. 不变边界

- 不修改现有全局导航栏。
- 不修改 `AI小高剪辑` 页面标题栏。
- 不修改“素材库 / 剪辑工作台”切换组件及路径。
- 不新增权限码，继续使用 `auto_wechat:ai_edit`。
- 不恢复已取消的一键过审能力。
- 不建设专业多轨编辑器、向量检索或自动抖音发布。
- 不把外部 `auto_edit` 仓库作为运行时依赖。
- Phase 13 完成前不进行宝塔生产验证。

## 5. 总体架构

```text
React 前端
  -> 9000：素材元数据、查询、分析结果、人工确认、云端预览
  -> 127.0.0.1:19000：本地导入、本地预览、分析执行、增稳、云端上传发起

19000 小高AI微信助手
  -> 本机受管素材目录
  -> 素材分析队列与重启恢复
  -> FFmpeg/ffprobe、ASR、场景切分、关键帧、稳定性检测
  -> 将必要分析摘要和上传文件流提交给 9000

9000 主服务
  -> PostgreSQL/SQLite 开发兼容的权威 metadata
  -> 宝塔受控素材根目录
  -> 调用 9100 内部素材语义分析接口

9100 AI 服务
  -> 严格输入输出的多模态语义分析
  -> 描述、分类、标签、高光和可用区间
```

### 5.1 前端职责

- 只发起用户操作并展示权威状态。
- 元数据和云端媒体走 9000；本地文件操作走 19000。
- 不持有 9100 内部令牌，不自报 `merchant_id`，不承担大文件中转。

### 5.2 19000 职责

- 作为本机原始素材文件真源。
- 自动执行本地分析阶段并持久化恢复状态。
- 只提交必要关键帧、转写与媒体摘要。
- 从本机受管文件流式上传到 9000，不把整个视频读入内存。
- 迁入 `auto_edit` 视觉分析时删除或隔离其报告中的 `source_path`；该字段只可用于本机临时诊断，禁止进入 9000、9100、公共响应或日志。

### 5.3 9000 职责

- 作为素材归属、生命周期、分析状态、人工确认和云端文件的权威源。
- 校验 Local Agent token 到商户映射、内容指纹、文件大小和幂等键。
- 维护受控相对存储键，公共响应不返回绝对路径、存储键或商户 ID。

### 5.4 9100 职责

- 只提供内部素材语义分析窄接口。
- 严格校验输入、输出、素材 ID 和时间区间。
- 失败只影响内容分析阶段，不触发伪造规则兜底。

## 6. 数据合同

迁移从当前 SQLite `0033` 与 PostgreSQL `0014` 之后继续，新增 SQLite `0034` 升降级脚本与 PostgreSQL `0015` Alembic 修订。历史迁移不得修改。

### 6.1 扩展 `ai_edit_materials`

新增安全展示和媒体字段：

- `display_name`
- `description`
- `category`
- `duration_seconds`
- `width`
- `height`
- `fps`
- `file_size_bytes`
- `manual_override_json`
- `manual_confirmed_at`
- `purge_operation_id`
- `purge_status`

保留现有 `thumbnail_storage_key`、`cloud_storage_key`、`storage_mode`、`analysis_status` 与 `stabilization_status`。旧状态字段作为兼容汇总，不再是前端分阶段状态的唯一事实。`purge_operation_id/purge_status` 只用于永久删除原子 claim；`purge_status` 只允许空、`preparing` 或 `completed`。claim 期间禁止恢复、新任务引用和第二次删除；completed tombstone 保留最后 operation ID，使 finalize 响应丢失后可幂等重放。

增加 `(merchant_id, source_sha256)` 唯一约束，阻断并发重复导入。平台素材 `merchant_id` 为空，不受该商户私有去重约束影响。

### 6.2 新增 `ai_edit_material_processes`

每个素材、源哈希和阶段一条当前状态记录：

```text
stage:
  media_probe
  transcript
  content_analysis
  stability
  cloud_upload

status:
  queued
  running
  succeeded
  failed
  not_required
```

字段包含 `progress`、`attempt_count`、随机 `execution_token_hash`、`failure_code`、脱敏 `error_summary`、`started_at`、`completed_at`、`updated_at`。`progress` 必须在 0 到 100 之间。阶段 claim 由 9000 单独接口执行，以当前源 SHA 和 `expected_attempt` 做 CAS，返回该阶段的新 attempt 与原始令牌；不同阶段使用独立令牌。重复注册既有素材不恢复旧原始令牌，只在需要执行时领取新 claim。状态回写必须同时匹配素材、源 SHA、阶段、attempt 和令牌，Local Agent token 本身不能替代 attempt 身份。

增加 `(material_id, source_sha256, stage)` 唯一约束，确保同一源版本每阶段只有一条当前状态；重试通过 `attempt_count` 推进，不插入并发状态副本。

### 6.3 延续 `ai_edit_material_analyses`

- 每次成功分析形成不可变版本快照。
- 保存源 SHA-256、分析版本、转写、场景、AI 标签和可用区间。
- JSON 入库前通过严格模型校验。
- 查询时使用最新有效快照，并与人工确认记录合并。

### 6.4 人工确认

人工分类、描述、标签和片段区间保存在素材的独立覆盖结构中。合并优先级为：

```text
人工确认 > 最新 AI 结果 > 空值
```

重新分析只创建新 AI 快照，不修改人工覆盖结构。

## 7. 接口设计

### 7.1 9000 商户接口

- `GET /ai-edit/materials`：支持 `scope`、`lifecycle`、关键词、分类、标签、五阶段状态、时长、创建时间、排序和分页。
- `GET /ai-edit/materials/{material_id}`：返回素材详情、有效分析结果和分阶段状态。
- `GET /ai-edit/materials/{material_id}/thumbnail`：登录态下返回安全缩略图，不暴露存储键。
- `GET /ai-edit/materials/{material_id}/content`：同源鉴权的云端分段预览。
- `PATCH /ai-edit/materials/{material_id}/annotations`：保存人工确认结果。

`lifecycle` 明确使用 `active` 或 `trash`，禁止继续在服务内固定过滤 `deleted_at IS NULL`。

平台公共素材管理使用 `/admin/ai-edit/materials` 管理接口，复用现有超管鉴权与 `auto_wechat:ai_edit` 能力边界，不新增权限码。普通商户调用管理接口一律拒绝，跨商户与平台归属信息不在错误消息中泄露。

### 7.2 9000 Local Agent 接口

延续 Local Agent token 鉴权，并增加：

- 按商户和 SHA-256 解析或注册规范素材 ID。
- 写入阶段状态和进度。
- 提交本地分析结果和缩略图。
- 流式上传云端文件。

本地分析提交采用严格 JSON DTO：总请求体不超过 8MB；最多 12 张关键帧；单帧 base64 解码后不超过 512KB、最大边不超过 720px，并校验 JPEG/WEBP 魔数；禁止任何路径字段。缩略图从已校验关键帧中选择并原子写入受控存储，数据库提交后才对外可见。
- 恢复、永久删除和到期清理协调。

所有写接口必须幂等，并拒绝旧 attempt 回写。

### 7.3 19000 接口

在现有 `/agent/ai-edit/materials` 下增加：

- 单素材重新分析。
- 单素材主动上传云端。
- 单素材一键增稳。
- 恢复与永久删除。
- 短时本地预览票据及分段媒体读取。

批量操作由前端以有限并发调用单素材接口，保留逐项结果，不新增难以恢复的批量事务。

### 7.4 9100 内部接口

- `POST /internal/ai-edit/materials/analyze`
- 复用既有内部服务鉴权。
- 请求只包含素材 ID、媒体摘要、场景时间、压缩关键帧和转写片段。
- 响应为严格结构化描述、分类、标签、高光、可用区间及置信度。

## 8. 核心数据流

### 8.1 导入与去重

```text
浏览器选择文件
  -> 19000 流式写临时文件并计算 SHA-256
  -> ffprobe 校验真实媒体
  -> 9000 按商户 + SHA-256 返回规范 material_id
  -> 19000 原子落入商户受管目录
  -> 9000 创建五阶段状态
  -> 19000 自动排队分析
```

如果 9000 已存在同哈希素材但当前电脑缺少文件，19000 使用规范素材 ID 补齐本机副本。重复素材不得创建第二条商户元数据。

升级前已存在的本地素材由 19000 启动协调逐项重登记 9000、采用规范 ID，并只对缺少当前 SHA 分析快照的素材领取阶段 claim 后排队；不得永久保留旧 `pending` 汇总状态。

### 8.2 自动分析

```text
media_probe
  -> transcript
  -> scene split + keyframes + stability
  -> 9000 保存本地结果
  -> 9000 调 9100 content_analysis
  -> 严格校验
  -> 版本化分析快照
```

本地阶段独立提交状态。9100 不可用时，内容分析标记失败，其余成功阶段保持可用。

### 8.3 云端上传

```text
用户主动上传
  -> 19000 从受管文件流式读取
  -> 9000 写入商户受控临时文件
  -> 校验大小与 SHA-256
  -> os.replace 原子提交
  -> storage_mode=cloud_available
```

受控路径结构由 9000 内部生成，数据库仅保存相对存储键。上传失败保持本地原文件不变，并将 `cloud_upload` 标为失败。

### 8.4 回收站

- 软删除后本地和云端副本均保留 7 天。
- 恢复由 19000 协调本地清单与 9000 metadata。
- 永久删除先由 9000 在数据库中原子写入删除 claim，再删除本机和云端受管副本。claim 期间禁止恢复、新任务引用和另一永久删除；finalize 必须匹配 claim 的 operation ID。
- 已完成永久删除的 tombstone 若因同商户同 SHA 再次导入而复活，9000 必须在同一事务清除删除字段、`purge_status` 和 `purge_operation_id`；旧 operation 的 finalize 随后只能返回中性冲突，不能删除新导入内容。
- 9000 到期后清理云端并记录待本机同步；19000 离线时在下次启动或心跳继续清理。
- 所有恢复和清理操作幂等。

### 8.5 预览

- 云端素材通过 9000 登录态和范围请求流式预览。
- 本地素材由 19000 签发短时、绑定商户和素材的预览票据。
- Local Agent token 不进入媒体 URL；票据过期或使用范围不匹配即拒绝。

## 9. 前端设计

### 9.1 冻结区域

以下现有区域与样式不修改：

- 全局导航栏。
- `AI小高剪辑` 标题栏。
- `ModuleTabs` 的“素材库 / 剪辑工作台”切换。

新增界面只替换素材范围切换下方的内容区，并复用当前前端的：

- 页面背景 `#f3f6fa`
- 边框 `#e4e8f0`
- 主文字与深色按钮 `#1a1f2e`
- 当前蓝色链接/激活色
- 当前绿色、琥珀色、红色状态色

### 9.2 桌面端

- 顶部保留“私有素材 / 平台公共 / 回收站”。
- 左栏为全部、口播、空镜、高光、未分类、智能标签和快捷状态。
- 中栏支持搜索、组合筛选、排序、网格/列表切换和批量选择。
- 右栏提供视频预览、媒体详情、五阶段状态、标签、时间轴和操作。

### 9.3 状态文案

- `queued`：待处理。
- `running`：显示真实阶段和进度。
- `succeeded`：已完成。
- `failed`：失败，可查看原因并重试。
- `not_required`：无需处理。
- `pending` 不得继续翻译为“处理中”。

素材卡只显示两个最高优先级状态；完整状态在右侧详情展示。

### 9.4 导入与批量操作

- 支持拖拽、多选文件和文件夹。
- 导入队列逐项展示校验、去重、写入、分析排队和失败重试。
- 单个失败不终止批次，重复素材可定位已有记录。
- 私有素材支持批量分析、上传云端、分类/标签和删除；只有视频显示“用于剪辑”，音频和图片本期不进入剪辑任务。
- 回收站支持批量恢复和永久删除。
- 平台公共素材不显示商户修改和删除入口。
- 已通过现有超管鉴权的管理员在“平台公共”范围看到导入、分析、编辑和删除入口；普通商户继续使用同一页面的只读形态。

### 9.5 详情与时间轴

- 展示真实视频、媒体参数、分析结果、稳定性和云端状态。
- 时间轴展示场景、高光、可用区间和不稳定区间。
- 人工调整后标记“人工确认”。
- 一键增稳生成衍生素材并保留来源关系。
- 视频用于剪辑时携带选中素材、角色和人工确认区间进入现有工作台；音频和图片本期只做素材库管理、分析、预览和上传，前端禁用入口，9000 创建任务与 Worker 合同再次拒绝，防止绕过前端扩展静帧/BGM 渲染协议。

### 9.6 移动端

移动端使用单栏；分类和筛选进入抽屉，详情全屏打开，底部保留核心批量操作。禁止把桌面三栏直接压缩到手机宽度。

## 10. 异常处理与恢复

- 导入采用临时文件、校验、9000 登记和原子落盘；失败不留下可见半成品。
- 分析阶段 attempt 持久化；19000 重启恢复未完成任务，旧 attempt 不得覆盖新结果。
- 上传中断时删除临时云端文件并保留本地原文件。
- 一期上传失败后从头重传，不引入分片续传；全程流式处理。
- 删除、恢复和永久清理均幂等。
- 本地解析失败时不启动依赖阶段。
- 转写失败时允许保留画面与稳定性结果，内容分析标记信息不完整。
- 9100 异常或拒答时不生成标签、高光或可用区间兜底。
- 云端上传失败时保持 `local_only`，不得显示“云端可用”。

## 11. 安全要求

- 19000 继续只监听 `127.0.0.1`，鉴权保持开启。
- 商户归属只来自登录态或 Local Agent token 映射。
- 本机与云端路径都必须位于商户受管根，拒绝穿越和符号链接。
- 上传校验真实媒体、大小和 SHA-256。
- 9100 只接受 9000 内部鉴权请求。
- 模型输入和输出执行提示词注入检查、严格结构校验、素材 ID 校验和时间范围校验。
- 日志不记录转写原文、关键帧内容、绝对路径、存储键或令牌。
- 公共响应继续脱敏 `merchant_id`、绝对路径、内部存储键和执行令牌。

## 13. 验收标准

- 回收站真实可见，支持恢复和永久删除。
- 导入后自动分析，五阶段状态最终收敛，不永久假“处理中”。
- 同商户重复文件不产生重复素材记录。
- 多模态失败不破坏本地分析结果。
- 单个和批量云端上传真实写入文件并校验。
- 原素材不因分析、增稳、上传或删除协调失败而被意外修改。
- 平台公共素材对普通商户只读。
- 人工确认结果不被重新分析覆盖。
- 桌面与移动端无重叠、白屏和文本溢出。
- 既有导航、标题栏、模块切换和剪辑工作台回归不变。
- 自动化不连接真实宝塔、生产数据库或真实付费模型。
- 同轮更新受影响的 Phase 12 专题文档和 `docs/ai/05_PROJECT_CONTEXT.md`。
