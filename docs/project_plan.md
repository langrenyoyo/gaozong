# 最终产品形态：

重点：明确产品主线为主系统的副系统

客户打开 React 前端；
auto_wechat系统后端负责：
自动拉取/接收线索；
线索入库（可能需要数据清洗）；
分配线索；
创建微信任务；
接收客户端执行结果；
更新线索状态；
React 展示状态；
客户电脑 B 运行开发提供的 exe / Local Agent；
客户电脑 B 上的 exe 负责：
操作本机微信；
搜索销售微信昵称；
粘贴/发送线索信息；
检测销售回复；
回写执行结果；
主系统根据 exe 回写结果更新 lead_notifications、reply_checks、douyin_leads.status。



# 主线流程

1.主系统自动从数据源拉取线索 → 线索清洗筛选后入库 → 线索分配给销售 → 创建微信发布线索任务 → 发给给客户本地电脑的local Agent/exe → 得到销售回复/超时 → 确认结果写回入库，对应线索任务状态更新为跟进中/确认为超时 → 对数据源进行回调反馈/线索重新分配，继续循环流程
2.客户本地电脑的local Agent/exe → 接受微信发布线索任务 → 微信搜索框输入对应微信昵称 → 确认为正确联系人 → 打开对应聊天窗口 → 粘贴线索信息 → 发送
3.客户本地电脑的local Agent/exe → 检测微信是否有销售信息 → 得到有效回复/超时 → 对结果进行反馈
