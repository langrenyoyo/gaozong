## 2. 产品目标

商户运营人员在安装小高AI微信助手的同一台 Windows 电脑上完成：

```text
导入汽车口播素材
  -> 素材分析与可选增稳
  -> 选择平台剪辑模板
  -> AI 生成保守剪辑计划
  -> 轻量人工调整
  -> 720P 预览
  -> 1080P 成片
  -> 本地下载或主动上传云端
```

一期只服务抖音汽车营销短视频：竖屏 `9:16`、目标时长 `15-60` 秒、中文字幕。成片不直接上传或发布到抖音。

## 3. 用户、权限与隔离

- 商户运营人员是主要用户。
- `auto_wechat:ai_edit` 同时控制“小高素材库”和“AI 小高剪辑”。
- 同商户且有权限的用户共享素材、任务和成片，并记录创建人和操作审计。
- 商户私有素材严格隔离；禁止商户间共享。
- 超管只维护平台公共素材、剪辑模板、配置和审计，不代替商户执行日常剪辑。
- `merchant_id` 只来自 9000 可信登录上下文或 Local Agent 鉴权映射，前端不得自报。

## 4. 一期非目标

- 不建设完整多轨时间线、关键帧、滤镜和专业音频编辑器。
- 不做 ROI 点选、框选或涂抹增稳。
- 不做远程设备控制和多设备管理。
- 不做商户间素材共享。
- 不做自动云同步和永久草稿版本历史。
- 不接抖音草稿上传、自动发布或网络素材抓取。
- 不建设向量语义检索；一期使用结构化标签和关键词。
- 不迁入来源项目的 PySide6、SQLite、本地 JSON 任务库、固定豆包客户端或开发 runner。
- 不调用外部 `auto_edit` 或 BrollStudio 仓库作为运行时依赖。

## 5. 总体架构

```text
React（与小高AI微信助手同机）
  -> 9000：登录、权限、商户 metadata、公共素材、任务、审计
  -> 127.0.0.1:19000：本地导入、预览、队列、进程监管

19000 小高AI微信助手（Python 3.10）
  -> 本地受管素材目录
  -> 默认单任务队列
  -> 启动/取消 ai_edit_worker.exe
  -> 与 9000 通信并转发安全进度

ai_edit_worker.exe（Python 3.11）
  -> ASR、分镜、标签、自动增稳
  -> 候选片段、字幕、BGM、FFmpeg 渲染
  -> 720P 预览、1080P 成片、媒体强校验
```

小高AI微信助手仍是一个产品和一个安装目录，但剪辑能力在独立子进程中运行。剪辑 Worker 缺失或损坏时只禁用 AI剪辑，不影响微信助手启动、轮询或安全门禁。

一期采用“一任务一子进程”。默认并发数为 `1`；只有实测启动和模型加载成本不可接受时，才升级为长驻 Worker。

## 6. 小高素材库

### 6.1 素材范围

- 商户私有素材：视频、图片、音频，默认仅本地保存。
- 平台公共素材：汽车空镜、片头、片尾和授权配乐，由超管维护，商户只读。
- 配乐只来自平台授权公共库或商户私有上传，禁止联网抓取。

### 6.2 本地导入

1. 浏览器可一次多选视频，为每个文件生成独立且不含本机路径的素材 ID。
2. 前端按选择顺序逐个把文件流式传给 19000；单个失败记录文件名和原因，不阻断后续文件。
3. 19000 将文件复制到受管目录，原文件只读且不被覆盖。
4. 19000 校验媒体类型、计算 SHA-256、执行 ffprobe 并生成低清缩略图。
5. 19000 向 9000 同步 metadata 和缩略图；9000 不保存本机绝对路径。
6. 19000 的本地清单把素材 ID 映射到受管目录相对路径；批次结束后前端只刷新一次 9000 素材列表。

### 6.3 存储状态

```text
local_only
  -> uploading
  -> cloud_available

local_only/cloud_available
  -> local_missing
  -> trashed
  -> purged
```

- 默认 `local_only`；用户主动选择后才上传云端。
- 仅本地素材在其他设备不可预览或剪辑。
- 平台公共素材固定为云端可用，19000 下载后必须复验大小和 SHA-256。
- 云端与本地只共享业务 metadata，不自动双向同步完整媒体。

### 6.4 分层分析

- 导入立即执行：哈希、媒体探测、缩略图。
- 用户主动分析或首次剪辑时执行：ASR、分镜、内容标签、稳定性评分、可用区间。
- 分析结果按源哈希、算法版本和模型版本复用。
- 原视频、原音频、图片和代理视频均不得发送给 9100。

### 6.5 删除

- 活动任务引用的素材禁止删除。
- 删除后进入 7 天回收站，可恢复。
- 到期后清理本地或云端受管副本；安全指纹和审计记录保留。

## 7. AI 小高剪辑

### 7.1 任务输入

- 至少一条主素材；业务层不限制素材数量、输入总时长或文件大小。
- 可选空镜、图片和授权配乐。
- 选择平台模板、目标时长和增稳选项。
- 无业务上限不等于无安全门：磁盘、临时空间、并发、阶段超时和重试次数必须可配置。

### 7.2 保守剪辑规则

- 主素材口播和画面保持同步。
- AI 只删除静音、口误、明显重复和无效片段。
- AI 保持主口播时间顺序，不生成原素材中不存在的车辆事实。
- 空镜只覆盖适合补画面的区间，继续使用主素材口播音频。
- AI 可生成标题、字幕和结尾行动引导建议。
- 9100 不可用时明确失败，不启用隐式规则兜底。

### 7.3 处理阶段

```text
preflight
  -> analyze
  -> stabilize_optional
  -> plan
  -> render_preview_720p
  -> review
  -> render_final_1080p
  -> verify
  -> completed
```

增稳只处理用户勾选或系统建议的素材，始终生成衍生文件并默认保留音频。BrollStudio 自动模式当前使用 `-an`，迁入时必须修正，不能静默丢音轨。

### 7.4 轻量调整

- 删除或调整片段顺序。
- 修改片段首尾时间。
- 替换空镜。
- 修改字幕文字、断句和时间范围。
- 开关字幕、配乐和指定素材增稳。

每次重新渲染替换当前草稿；用户确认后冻结成片。只保留当前草稿和已确认成片，不保留永久版本历史。

### 7.5 输出

- `720x1280` 快速预览。
- `1080x1920` 正式成片。
- H.264 视频、AAC 音频，帧率跟随主素材并收敛到常用范围。
- 成片默认保存在本机，用户可下载或主动上传云端。

## 8. 任务状态与恢复

```text
queued -> running -> review_required -> succeeded
queued/running/review_required -> cancel_requested -> cancelled
queued/running -> failed
```

- 19000 默认单任务执行，其余排队。
- 取消必须终止 Worker 和 FFmpeg 进程树。
- 重启时把未完成的 `running` 任务重新入队，复验阶段产物后从最近安全阶段继续。
- 人工重试创建新 attempt，旧进程和旧令牌不能覆盖新结果。
- 进程退出但没有合法结果文件时记为失败，不伪造成功。

每个任务使用独立目录：

```text
ai_edit/jobs/{job_id}/{attempt_id}/
  manifest.json
  input/
  stages/
  output/
  progress.jsonl
  result.json
  diagnostics/
```

任务目录和路径全部由 19000 生成，前端不能提交绝对路径。

## 9. 9100 与算力

- Worker 不直接访问 9100，也不持有模型令牌。
- 19000 经 9000 调用 9100，只发送转写文本、镜头标签、时长和稳定性摘要。
- 9100 输出严格、版本化的剪辑计划 JSON；未知字段、非法区间和越界值必须拒绝。
- 仅 9100 剪辑规划、标题和行动引导计入现有 `compute` 算力。
- 本地 ASR、FFmpeg、分镜和增稳不扣算力。
- 原始模型响应不得进入普通日志、商户 artifact 或前端响应。

## 10. 数据模型方向

复用并扩展现有 `AiEditJob`、`AiEditJobArtifact`，不另建第二套任务表。

新增最少对象：

- `AiEditMaterial`：素材归属、媒体属性、设备、存储和生命周期状态。
- `AiEditMaterialAnalysis`：版本化 ASR、分镜、标签、稳定性和可用区间。
- `AiEditTemplate`：平台模板、剪辑规则和 Prompt 版本。
- `AiEditJobMaterial`：任务素材角色、顺序、固定哈希和使用区间。

扩展任务壳：状态、阶段、进度、设备 ID、attempt、执行令牌哈希、取消/心跳、输入指纹、引擎/模板/模型版本和稳定失败码。

扩展产物壳：本地/云端位置类型、设备 ID、SHA-256、大小、媒体属性、完整性状态和来源产物 ID。外部 API 不返回绝对路径、`storage_key`、执行令牌或 `merchant_id`。

一期直接复用现有 `agent_client_id` 绑定本地素材，不新建多设备管理系统。

```text

/ai-edit/templates

/ai-edit/jobs/{id}/plan
/ai-edit/jobs/{id}/confirm
```

```text

/agent/ai-edit/materials/{id}/stabilize
```

## 12. 前端设计

### 12.1 小高素材库

- 私有素材、平台公共素材、回收站三个标签页。
- 导入、搜索、媒体类型、标签、分析状态和稳定性筛选。
- 素材项展示真实缩略图、时长、存储位置和处理状态。
- 详情提供预览、标签、可用区间、分析、增稳、上传云端和删除。
- 公共素材只读，下载并校验后才能参与本地剪辑。

### 12.2 AI 小高剪辑

- 素材选择、平台模板、目标时长和增稳选项。
- 真实任务阶段、进度、取消和失败重试。
- 720P 草稿的片段、空镜、字幕和配乐轻量调整。
- 1080P 成片确认、预览、下载和主动上传云端。
- 不显示假任务、假素材、假统计或伪成功状态。

页面刷新后先从 9000 恢复业务状态，再向 19000 查询本地执行状态。浏览器与小高AI微信助手必须运行在同一台电脑。

## 13. 安全与错误处理

- 拒绝路径逃逸、符号链接、伪装扩展名和不可探测媒体。
- 资源不足在任务启动前阻断，不能边处理边耗尽磁盘。
- 日志只记录任务 ID、素材 ID、哈希前缀、阶段、进度、`failure_stage` 和稳定错误码。
- 原素材、转写原文、绝对路径、密钥和模型原始响应不得写入普通日志。
- 所有有效阶段产物必须钉住输入哈希、参数摘要、算法版本和媒体探测结果。
- 任一媒体质量门禁失败时，不得注册为可下载成片。

## 15. 验收

### 15.1 自动化

- 纯逻辑、模型、状态机、权限、商户隔离和迁移测试。
- 19000 路径安全、磁盘门禁、队列、取消和重启恢复测试。
- 使用合成媒体执行 Vid.Stab、裁切拼接、字幕、BGM 和双分辨率渲染。
- LLM、Embedding 和外部网络全部使用替身。
- 前端接口合同、离线状态、刷新恢复、TypeScript 和构建通过；9000 标准 `{success,data,message}` 响应经 `apiClient` 拦截器处理后只解包一层 `data`。
- 素材库批量导入合同通过：文件多选、双文件顺序流式导入、批次内素材 ID 唯一、单个失败继续和批次结束只刷新一次列表。

### 15.2 媒体强门

- 原素材 SHA-256 不变。
- 输出可被 ffprobe 读取且非空。
- 分辨率、时长、帧率、音频和时间戳符合任务合同。
- 字幕和口播不漂移，空镜不冻结尾帧。
- 增稳结果保持时长和音频，不覆盖原素材。

### 15.3 本地真实素材验收

在普通 Windows CPU 电脑上，用已授权的真实汽车口播素材完成完整单任务闭环。9100 使用替身；真实付费模型、生产数据库、宝塔和抖音发布统一留到 Phase 13 后的生产验证执行包。

### 15.4 甲方测试交付

检查点 C 通过后新增甲方测试专用单入口 EXE 交付。甲方只接收并启动 `小高AI系统测试版.exe`；外层控制进程使用 PyInstaller 自身临时目录承载 Local Agent 与 AI剪辑 Worker 两套隔离运行时，不额外维护 payload 清单或版本释放器。测试包只用于验证当前本地 MVP，不等同于生产验收；商户测试令牌不进入构建产物，Worker 不继承该凭据。按用户决定，许可证文件、组件清单、FFmpeg 构建信息和安全软件扫描均不作为 Task 11 开发、构建或测试交付的阻断条件。
